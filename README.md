# GraphZ - веб-редактор изображений

Учебный проект по дисциплине «Технологии компьютерной графики».

- **Лабораторная работа 1**: загрузка и отображение изображений на `<canvas>`,
  скачивание в форматах **PNG**, **JPG** и собственном формате курса
  **GrayBit-7 (GB7)**.
- **Лабораторная работа 2**: панель цветовых каналов и инструмент «Пипетка»
  с показом координат, RGB и **CIELAB**.
- **Лабораторная работа 3**: инструмент «Уровни» (Levels) с гистограммой,
  тремя ползунками входных уровней (чёрная точка, гамма, белая точка),
  предпросмотром в реальном времени и применением через LUT.
- **Лабораторная работа 4**: масштабирование изображения, контрол масштаба
  отображения и инструмент «Размер» с алгоритмами интерполяции
  (ближайший сосед, билинейная).

## Возможности

### Работа с файлами (Лаб. 1)

- Загрузка изображений `.png`, `.jpg/.jpeg` (средствами браузера) и `.gb7`
  (собственная реализация декодера).
- Отрисовка загруженного изображения на HTML5 `<canvas>` с сохранением пропорций
  и фоном-шахматкой для прозрачных областей.
- Скачивание изображения в форматах PNG, JPG и GB7 (собственная реализация кодера).
- Строка состояния с исходными шириной, высотой и глубиной цвета.
- Адаптивная вёрстка: холст пересчитывается при изменении размеров окна.

### Цветовые каналы и пипетка (Лаб. 2)

- Боковая панель «Каналы» с миниатюрами каждого канала загруженного изображения.
  Состав каналов определяется автоматически по формату:
  - GB7 без маски: 1 канал (Серый);
  - GB7 с маской: 2 канала (Серый + Альфа);
  - PNG/JPG без альфы: 3 канала (R, G, B);
  - PNG/JPG с альфой: 4 канала (R, G, B + Альфа).
- Миниатюры цветных каналов окрашены в свой цвет (красный, зелёный, синий),
  альфа-канал показывается в градациях серого как маска прозрачности.
- Клик по миниатюре включает или выключает канал. Активные каналы выделены
  рамкой, неактивные затемнены.
- Изменение состояния каналов мгновенно отражается на главном холсте.
  Если оставить только альфу, изображение покажется как ч/б маска прозрачности.
- Оригинальный массив пикселей не перезаписывается, для отрисовки каждый раз
  создаётся новый буфер RGBA.
- Инструмент «Пипетка» включается кнопкой в тулбаре. По клику на изображение
  в боковой панели показываются:
  - координаты `X, Y` пикселя относительно исходного изображения (с учётом
    масштабирования холста);
  - значения `R, G, B` (0..255) и `Alpha`, если он не равен 255;
  - HEX-код цвета;
  - координаты в пространстве **CIELAB** (D65).

### Уровни и гистограмма (Лаб. 3)

- Открывается кнопкой «Уровни» в тулбаре, в виде модального диалогового окна.
- Выпадающий список выбирает режим коррекции:
  - **Master (RGB)** — общая (композитная) гистограмма по светлоте
    `Y = 0.299·R + 0.587·G + 0.114·B`, действие применяется ко всем RGB каналам;
  - **Красный / Зелёный / Синий / Альфа** — гистограмма и коррекция отдельного
    канала. Для GB7 с маской доступны Master и Альфа.
- Гистограмма строится самостоятельно на `<canvas>` (256 столбцов, ось 0..255).
- Переключатель `log` меняет ось ординат: линейная или логарифмическая
  (`log(1 + n)`).
- Под гистограммой ползунок с тремя ручками: чёрная точка, гамма (полутона),
  белая точка. Дополнительно поля для точных значений: чёрная 0..253,
  гамма 0.1..9.9, белая 2..255. Маркеры не пересекаются, гамма всегда между ними.
- Связка гаммы и положения средней ручки:
  `midRel = (mid − black) / (white − black)`,
  `gamma = log(midRel) / log(0.5)`. Сдвиг средней ручки влево осветляет средние
  тона, вправо — затемняет.
- Изменения для Master и каждого канала хранятся отдельно и сохраняются при
  переключении в выпадающем списке.
- Чекбокс «Предпросмотр на холсте» по умолчанию включён. При движении ползунков
  результат пересчитывается через LUT и применяется к изображению на основном
  холсте на следующем кадре (`requestAnimationFrame`). Снятие чекбокса
  моментально возвращает оригинал «до».
- **Сброс** возвращает все четыре набора параметров к значениям по умолчанию,
  диалог не закрывается.
- **Отмена** закрывает диалог, отбрасывает все изменения, возвращает холст
  к состоянию до открытия инструмента.
- **Применить** формирует таблицы подстановки (LUT) для каждого канала, один
  раз перезаписывает данные изображения и закрывает диалог.

### Масштабирование (Лаб. 4)

- При загрузке изображение автоматически вписывается в холст с отступом
  не менее 50 пикселей с каждой стороны. Итоговый зум ограничен диапазоном
  `12% .. 300%`.
- В строке состояния доступен контрол масштаба:
  - кнопка «Fit» — вписать в холст с отступом;
  - выпадающий список с пресетами `12, 25, 50, 75, 100, 150, 200, 300%`;
  - текстовое поле для произвольного значения, применение по `Enter` или
    потере фокуса; ввод валидируется и зажимается в допустимый диапазон.
- Само отображение масштабируется средствами браузера (`drawImage`), пиксели
  оригинала не пересчитываются. Это даёт быстрый отклик и плавное
  изменение масштаба.
- Инструмент «Размер» (кнопка в тулбаре, открывает модальный диалог) меняет
  фактические размеры изображения с пересчётом пикселей:
  - блоки «До» и «После» показывают ширину × высоту и количество мегапикселей;
  - выпадающий список «Единицы»: проценты или пиксели. При смене единиц
    значения автоматически пересчитываются;
  - поля «Ширина» и «Высота» с числовым вводом и валидацией диапазона
    (`1..1000%`, `1..8192 px`);
  - чекбокс «Сохранять пропорции» жёстко связывает оба поля по исходному
    отношению сторон;
  - выпадающий список алгоритмов интерполяции с подсказкой по каждому
    (иконка `?`):
    - **Ближайший сосед** — берётся ближайший пиксель оригинала. Самый
      быстрый, без сглаживания, сохраняет резкие края (полезно для пиксельной
      графики и масок).
    - **Билинейная** (по умолчанию) — взвешенное среднее четырёх соседних
      пикселей. Гладкое масштабирование без явных ступенек.
- Архитектура расширяема: новые алгоритмы добавляются в массив `ALGORITHMS`
  и реестр `RESAMPLERS` файла `src/resample.tsx` без изменений UI.
- После применения создаётся новое изображение, текущий зум пересчитывается
  как «Fit» от новых размеров, состояние пипетки и каналов обновляется.

## Формат GrayBit-7

Несжатый формат изображений в оттенках серого, 7 бит на пиксель плюс опциональный бит маски.

| Смещение | Размер | Описание |
|---------:|-------:|----------|
| 0 | 4 | Сигнатура `47 42 37 1D` |
| 4 | 1 | Версия (`0x01`) |
| 5 | 1 | Флаги (бит 0: наличие маски) |
| 6 | 2 | Ширина (uint16, big-endian) |
| 8 | 2 | Высота (uint16, big-endian) |
| 10 | 2 | Резерв (`0x0000`) |
| 12 | W*H | Данные пикселей |

Каждый байт пикселя: биты 6..0 это значение серого (0..127), бит 7 это маска
(1 - пиксель непрозрачный, 0 - прозрачный, если флаг маски установлен).

Реализация декодера и кодера: `src/gb7.tsx`.

## Технологии

- [Vite](https://vitejs.dev/) + [React](https://react.dev/) + TypeScript
- [Material UI](https://mui.com/): библиотека компонентов интерфейса

Работа с пикселями, форматом GB7, преобразованием sRGB в CIELAB и манипуляции
с каналами реализованы самостоятельно, без сторонних графических библиотек.

## Структура проекта

```
src/
  gb7.tsx           кодер/декодер формата GrayBit-7
  imageIO.tsx       загрузка/сохранение PNG, JPG, GB7
  color.tsx         sRGB -> XYZ -> CIELAB (D65)
  channels.tsx      список каналов, миниатюры, применение маски
  levels.tsx        параметры уровней, LUT, применение коррекции
  histogram.tsx     подсчёт и отрисовка гистограммы на canvas
  resample.tsx      алгоритмы интерполяции и функция изменения размера
  types.tsx         общие типы (PixelImage, ImageMeta, ToolMode, PixelSample)
  ImageCanvas.tsx   отрисовка изображения на canvas, клик пипетки, зум
  SidePanel.tsx     боковая панель (каналы + пипетка)
  LevelsDialog.tsx  диалог «Уровни»: гистограмма, ползунки, кнопки
  ResizeDialog.tsx  диалог «Размер»: пиксели/проценты, пропорции, алгоритм
  StatusBar.tsx     строка состояния и контрол масштаба
  Toolbar.tsx       панель инструментов (загрузка, скачивание, инструменты)
  App.tsx           композиция приложения
samples/          тестовые изображения .gb7
```
